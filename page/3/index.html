Swift-基本运算符

发表于 2019-05-27 | 分类于 iOS开发

赋值运算符

  1. 赋值的右边是一个多元组，它的元素会被分解多个常量和变量。

  2. Swift的赋值操作并不返回任何值。C和OC是返回值得。

算术运算符

  1. Swift默认情况下不允许在数值运算中出现溢出的情况。

组合赋值运算符

  1. let b = a += 2是错误的，因为复合赋值运算没有返回值，a += 2是没有返回值的。

比较运算符

  1. Swift 也提供恒等（===）和不恒等（!==）这两个比较符来判断两个对象是否引用同一个对象实例。

  2. 元组的元素相同，且长度相同，元组就可以被比较。元组的比较会从左到右逐个比较，知道发现有两个值相等时停止。

  3. Bool不能被比较，所以含有布尔类型的元组不能被比较。

  4. Swift 标准库只能比较七个以内元素的元组比较函数。如果你的元组元素超过七个时，你需要自己实现比较运算符。

空合运算符

  1. 空合运算符(a ?? b)将对可选类型a进行空判断，如果a包含一个值就解包，否则返回一个默认值b。表达式 a 必须是 Optional 类型。默认值 b 的类型必须要和 a 存储值的类型保持一致。

区间运算符

  1. 闭区间运算符(a...b)定义一个从a到b的所有值的区间。

  2. 半开区间运算符(a..<b)定义一个从a到b但不包含b的区间。

单侧区间

  1. 闭区间操作符有另一表达形式，表达往一侧无限延伸的区间。

  2. 半开区间操作符也有单侧表达式。

  3. 不能遍历省略了初始值的单侧区间，因为遍历的开端并不明显。

Swift-基础部分

发表于 2019-05-24 | 分类于 iOS开发
  1. Siwft包含C和OC上所有的基本数据类型。还有三个基本的几个类型，Array、Set、Dictionary。
  2. 常量：值不可改变的变量。

  3. 元祖（Tuple）可以创建或着传递一组数据，是Swift的高阶数据类型。元祖内的值可以是任意类型的，并不要求是相同类型。

  4. Swift用可选类型（Optional）来处理值缺失的情况，可用在任何类型上。

  5. Swift是一门类型安全的语言。在编译时进行代码的类型检查，把不匹配的类型标记为错误。
  6. 常量的值一旦设定就不能改变，变量的值可以随意更改。
  7. 用let声明常量，var声明变量。

  8. 声明常量或者变量的时候可以加上类型标注，说明常量或者变量中要存储的值的类型。例子:

  9. Swift不用语句结尾处添加分号;,在同一行内写多条独立的语句时必须使用分号分隔。

  10. 可以使用不同整数类型的min和max属性来获取最小值和最大值。

  11. Int类型在32位平台上和Int32长度相同，在64位平台上和Int64长度相同。

  12. Double表示64位浮点数，需要高精度浮点数时使用。Float表示32位浮点数。
  13. 通过给变量和常量在声明时赋初始值，让编译器在编译代码的时候可以自动推断出变量和常量的类型，这就是Swift的类型推断。
  14. 推断浮点数的类型时，Swift总是会选择Double。

  15. SomeType(ofInitialValue)是调用Swift构造器并传入一个初始值的默认方法。可以用来类型转换。

  16. 整数和浮点数之间的转换必须显示指定类型。

  17. 结合数字类常量和变量不同于结合数字类字面量。数字字面量本身没有类型，它的类型只在编译器需要求值的时候被推测。

  18. 类型别名就是给现有类型定义另一个名字。使用关键字typealias来定义。

  19. Bool是Swift的布尔类型，值是true和 false。

可选类型

  1. 使用可选类型来处理值可能缺失的情况。
  2. Int?表示一个可选Int，表示可能包含值，也可能不包含值。
  3. 可选变量可以被赋值为nil来表示没有值。
  4. nil不能用于非可选的变量和常量。
  5. 声明的可选常量和变量如果没有初始值，自动被置为nil。
  6. Swift和OC中的nil是不一样的。OC中nil指向一个不存在对象的指针。·Swift中nil是一个确定的值，用来表示值缺失，任何类型的可选状态都可以被置为nil。
  7. 当确定一个可选类型确实包含值，可以使用!来获取值，这被称为可选值的强制解析。

  8. 使用可选绑定来判断可选类型是否包含值，如果包含就把值赋给一个临时的变量或者常量。在if和while语句中使用可选绑定，可以判断可选类型中是否有值。

  9. 在if语句中使用可选绑定将值赋给常量或者变量，仅在if语句的句中才能获取到值。

错误处理

  1. 使用错误处理来应对程序中可能遇到的错误，错误处理可以推断失败的原因，并传播至程序的其他部分。

  2. 一个函数可以在通过声明中添加throws关键字来抛出错误消息，调用函数时应该使用try关键字。一个do语句创建了一个新的包含作用域，错误可以被传播到一个或多个catch从句。

断言和先决条件

  1. 断言和先决条件是在运行时所作的检查，用来检查在执行后续代码之前是否一个必要的条件已经被满足。

  2. 断言和先决条件的不同点：断言仅在调试环境运行，而先决条件则在调试环境和生产环境中运行。

  3. 可以在开发阶段使用断言，这些断言在生产环境中不会产生任何影响。

  4. 使用assert(_:_:file:line:)函数来写一个断言。

  5. 使用precondition(_:_:file:line:)函数写一个先决条件。
