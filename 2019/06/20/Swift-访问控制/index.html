Swift-访问控制

发表于 2019-06-20 | 分类于 iOS开发

  1. 访问控制可以限定其他源文件或模块中的代码对你的代码的访问级别。

  2. 模块指的是独立的代码单元，框架或应用程序会作为一个独立的模块来构建和发布。

  3. 一个模块可以使用import关键字导入另外一个模块。

  4. 源文件就是Swift中的源代码文件，它通常属于一个模块，即一个应用或者框架。

  5. Swift提供了五种不同的访问级别：

    • Open和Public级别可以让实体被同一模块中源文件中的所有实体访问，在模块外也可以通过导入该模块来访问源文件里的所有实体。
    • Internal级别让实体被同一模块中源文件的所有实体访问，但不能被模块外的实体访问。
    • File-private级别让实体只能在其定义的文件内部访问。
    • Private限制实体只能在其定义的作用域，已经同一文件内的extension访问。

  6. Open只能作用域类和类的成员，它和Public的区别如下：

    • Public或其他更严访问级别的类，只能在其定义的模块内部被继承。
    • Public或其他更严访问级别的类成员，只能在其定义的模块内部的子类中重写。
    • Open的类，可以在其定义的模块中被继承，也可以在引用它的模块中被继承。
    • Open的类成员，可以在其定义的模块中的子类中重写，也可以在引用它的模块中的子类中重写。

  7. Swift中的访问级别遵循一个级别准则：实体不能定义在具有更低级别的实体中。

  8. Swift中实体的默认访问级别是internal。

  9. 如果在导入应用程序模块的语句前使用 @testable 特性，然后在允许测试的编译设置（Build Options -> Enable Testability）下编译这个应用程序模块，单元测试目标就可以访问应用程序模块中所有内部级别的实体。

  10. 通过修饰符open、public、internal、fileprivate、private来声明实体的访问级别。

  11. 一个public类型的所有成员的访问级别默认为internal级别。

  12. 元组的访问级别是在它被使用时候自动推断出来的，元组的访问级别将由元组中访问级别最严格的类型来决定。

  13. 函数的访问级别根据参数类型或返回类型中最严格的访问级别来决定，如果推断出的访问级别与函数所在环境的默认访问级别不一致，需要明确指定函数的访问级别。

  14. 枚举成员的访问级别和该枚举类型相同。枚举定义中的任何原始值或关联值的类型的访问级别不能低于枚举类型的访问级别。

  15. 子类的访问级别不能高于父类的访问级别。

  16. 子类可以通过重写为继承来的类成员提供更高的访问级别。

  17. 常量、变量、属性、下标的Getters和Setters的访问级别和它们所属类型的访问级别相同。

  18. Setter的访问级别可以低于Getter的访问级别。

  19. 必要构造器必须和所属类型的访问级别相同。

  20. 一个类型被指定为public，默认构造器的访问级别是internal。

  21. 协议中的每一个要求和该协议是相同的访问级别，无法改变。

  22. Extension成员具有和原始类型成员一致的访问级别。

  23. 使用Extension来遵循协议，就不能显示的声明extension的访问级别。

  24. Extension的私有成员：

    • 在类型里声明一个私有成员，在同一文件的extension里访问。
    • 在extension里声明一个私有成员，在同一文件的另一个extension里访问。
    • 在extension里声明一个私有成员，在同一文件的类型声明里访问。
