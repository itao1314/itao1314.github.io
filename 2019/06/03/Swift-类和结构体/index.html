Swift-类和结构体

发表于 2019-06-03 | 分类于 iOS开发

类的实例被称为对象。

结构体和类对比

类和结构体的相同点：

  • 定义属性用于存储值

  • 定义方法用于提供功能

  • 定义下标操作用于通过下标语法访问它们的值

  • 定义构造器用于设置初始值

  • 通过扩展以增加默认实现之外的功能

  • 遵循协议以提供某种标准功能

类和结构体的不同点：

  • 继承允许一个类继承另一个类的特征

  • 类型转换允许在运行时检查和解释一个类实例的类型

  • 析构器允许一个类实例释放任何其所分配的资源。

  • 引用计数允许对一个类的多次引用

  • 结构体有一个自动生成的成员逐一构造器，类没有

类型定义的语法

  1. struct关键字定义结构体，class关键字定义类。

  2. 每当定义一个新的结构体或者类时，都是定义了一个新的Swift类型。

  3. 存储属性是与类或结构体绑定的，并存储在其中的常量或变量。

  4. 所有的结构体都有一个自动生成的成员逐一构造器，用于初始化新结构体实例中成员的属性。

结构体和枚举是值类型

  1. 值类型是这样一种类型，当它被赋值给一个变量、常量或者被传递给一个函数的时候，其值会被拷贝。

  2. Swift 中所有的基本类型：整数（integer）、浮点数（floating-point number）、布尔值（boolean）、字符串（string)、数组（array）和字典（dictionary），都是值类型，其底层也是使用结构体实现的。

  3. 结构体和枚举的实例，以及实例中所包含的任何值类型的属性，在代码传递的时候都会被复制。

  4. Swift对值类型的复制进行了优化，新的值类型不会立刻被复制,而是和原值类型共享一份内存，在某个副本要被修改前，才会进行复制。

类是引用类型

  1. 引用类型在被赋值到一个变量、常量或被传递到一个函数时，其值不会被复制，使用的是已经存在实例的引用，而不是拷贝。

  2. 类的实例被声明为常量，仍然可以修改属性值。因为声明的常量只是对类的实例的引用，改变类的实例的属性并没有修改声明的常量的引用值。

恒等运算符

  1. 判断两个常量或变量是否引用同一个类实例用恒等运算符：相同(===)和不相同(!==)。

  2. 相同(===)和等于(==)是不同的，相同(===)表示两个常量或变量引用同一个类实例，等于(==)表示两个实例的值相等或等价。

  3. Swift 中引用了某个引用类型实例的常量或变量，与 C 语言中的指针类似，不过它并不直接指向某个内存地址，也不要求你使用星号（*）来表明你在创建一个引用
